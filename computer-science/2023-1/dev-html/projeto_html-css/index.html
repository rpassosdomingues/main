Sistema de Controle de Versões Git e Repositório GitHub

GitHub Git Tutoriais Git

Descrição

O Git é um sistema de controle de versões distribuído amplamente utilizado para rastrear mudanças em projetos de desenvolvimento de software. Ele permite que várias pessoas colaborem em um projeto, rastreiem alterações, resolvam conflitos e revertam para versões anteriores do código. O GitHub é um serviço de hospedagem de repositório baseado em Git que fornece uma interface amigável para trabalhar com projetos versionados pelo Git.

Comandos Git

Aqui estão alguns comandos Git importantes:

  • init: Inicializa um repositório Git em um diretório.
  • clone: Clona um repositório Git existente para uma nova pasta.
  • branch: Lista, cria ou exclui branches (ramificações) do projeto.
  • checkout: Altera o branch atual e restaura arquivos do repositório.
  • status: Mostra o status atual do repositório, incluindo arquivos modificados, adicionados ou excluídos.
  • diff: Mostra as diferenças entre versões de arquivos.
  • add: Adiciona arquivos ao repositório para serem commitados.
  • commit: Registra as alterações no repositório.
  • push: Envia as alterações para um repositório remoto (como o GitHub).
  • pull: Obtém as alterações mais recentes de um repositório remoto e mescla com o branch atual.
  • revert: Desfaz um commit anterior, criando um novo commit para reverter as alterações.
  • merge: Combina as alterações de um branch com outro.
  • stash: Armazena temporariamente as alterações não commitadas para permitir a mudança de branch.
  • rm: Remove arquivos do repositório.
  • config: Configura opções do Git.
  • reset: Desfaz alterações específicas no repositório.
  • remote: Gerencia conexões remotas com repositórios.
  • fetch: Obtém as alterações mais recentes de um repositório remoto sem mesclar com o branch atual.
  • show: Mostra informações sobre objetos Git.
  • help: Exibe ajuda para um comando Git específico.
  • rebase: Reaplica commits de um branch em cima de outro.
  • blame: Mostra quem modificou cada linha de um arquivo.
  • tag: Marca versões específicas do repositório para facilitar referências futuras.

Configuração e Acesso ao Repositório GitHub

  1. Crie uma conta no GitHub em https://github.com/.
  2. Crie um novo repositório no GitHub.
  3. No seu terminal, navegue até o diretório do seu projeto e execute o comando git init para inicializar um repositório Git local.
  4. Use o comando git remote add origin <URL_do_repositório_GitHub> para adicionar o repositório remoto ao seu repositório Git local.
  5. Utilize os comandos Git para adicionar, modificar e commitar os arquivos do projeto.
  6. Para enviar suas alterações para o repositório remoto no GitHub, use o comando git push origin <branch>.

Utilização do Git/GitHub no Projeto

Durante a construção das páginas HTML deste projeto, foi utilizado o Git e o GitHub para controlar as versões dos arquivos e permitir o auto-aprendizado de uma futura colaboração eficiente. Foram adicionados e modificados todos os arquivos do projeto no repositório GitHub utilizando os comandos Git mencionados acima.
